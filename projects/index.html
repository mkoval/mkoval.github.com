Hardware Abstraction for Vex (HAX)

Michael Koval and Cody Schafer (Github)

Hardware Abstraction Layer (HAL) being developed by Rutgers' IEEE Robotics Vex Division to aid the team's transition from the old PIC-18 based microprocessor to the ARM Cortex M3-based microprocessor. For more detailed information, see the documentation hosted on the project's Github wiki.

IR Utilities

Michael Koval (Github)

ir-utils is a large collection of Python scripts written to collect, parse, and interpret data from a Sharp IR Ranger (specifically, the GP2D12 mid-range sensor). These scripts requires pySerial for data collection script (ir-nlines.py), NumPy for the data interpretation (liveaverage.py and liveinterp.py), and an installation of gnuplot in your PATH for live plotting (liveplot.py).

These scripts are designed to process data in real-time and can be chained together using Unix pipes, allowing for this small collection of scripts to serve a great variety of roles. For example, the following shell command will create a real-time, (mostly) noise-free plot of distance (in centimeters) between the front of the sensor and the nearest obstacle:

./ir-nlines.py /dev/cu.PL2303-00001004 all \ | ./liveaverage.py mean 10 2 \ | ./liveinterp.py IRCurve.txt 2 \ | ./liveplot.py -n500 --min 0 --max 150 1 2

where /dev/cu.PL2303-00001004 is the serial device from which the data is being received and IRCurve.txt is a CSV file containing distance → reading pairs that are representative of the electronic properties of the sensor (i.e. collected during calibration).

All testing was done on a computer running Mac OS X 10.6 (aka Snow Leopard) using the PIC-18 Vex Microprocessor v1.5 and the official Vex USB-to-Serial programming cable. That being said, all ir-utils scripts are designed to be hardware agnostic and should function identically with any infrared proximity sensor on a Unix-like operating system.

NetBots: Swarm Robotics Target Recognition Algorithm

Michael Koval, Adina Rubinoff, Mahsa Maghami, and Professor Michael Georgiopolous in the 2009 AMALTHEA REU

NetBots is a novel swarm robotics algorithm to improve detection accuracy in the distributed automatic target recognition (ATR) problems. Prior works (Dasgupta et al. and Martinoli et al.) have applied swarm robotics to the ATR problem by having a large number of cheap, inaccurate robots "vote" on whether a given object is a target.

These prior works, however, assume that the host of inexpensive robots is homogeneous in detection accuracy (i.e. all robots have the same probability of misidentifying a target). This is an inaccurate assumption: inexpensive production techniques are error-prone and produce robots of widely varying accuracy. The NetBots algorithm uses a decentralized social network of robot interactions to approximate each robots' individual accuracy. With this additional information, the robots' votes can be weighted to give more accurate robots greater influence when classifying a potential target.

Initial results show a significant decrease in the number of errors experienced by the NetBots algorithm when compared to the control HelpBots algorithm. This research was submitted to the IEEE ICRA-2010 conference for review.
